# 📋 Configuración de Workspace para Cursor

## ⚠️ PROBLEMA ACTUAL

El workspace de Cursor está abierto en un **worktree** (`pfo`) en lugar del repositorio principal.
Esto causa que los cambios se hagan en el worktree en lugar de `main`.

## ✅ SOLUCIÓN

### Opción 1: Cambiar el Workspace en Cursor (RECOMENDADO)

1. **Cerrar el workspace actual:**
   - File → Close Workspace (o Ctrl+K, luego Ctrl+W)

2. **Abrir el repositorio principal:**
   - File → Open Folder (o Ctrl+K, luego Ctrl+O)
   - Navegar a: `C:\Users\exclu\erp-taller-saas`
   - Seleccionar la carpeta y hacer clic en "Select Folder"

3. **Verificar que estás en main:**
   - En la terminal de Cursor, ejecutar: `git branch --show-current`
   - Debe mostrar: `main`

### Opción 2: Configurar Workspace Settings

Crear archivo `.vscode/settings.json` o `.cursor/settings.json`:

```json
{
  "git.autoRepositoryDetection": true,
  "git.detectSubmodules": false,
  "git.ignoreSubmodules": true,
  "git.defaultCloneDirectory": "C:/Users/exclu/erp-taller-saas",
  "files.watcherExclude": {
    "**/.cursor/worktrees/**": true,
    "**/.git/worktrees/**": true
  }
}
```

### Opción 3: Eliminar el Worktree Actual (SI NO LO NECESITAS)

```bash
# Desde el repositorio principal
cd C:/Users/exclu/erp-taller-saas
git worktree remove C:/Users/exclu/.cursor/worktrees/erp-taller-saas/pfo --force
```

## 🔍 VERIFICACIÓN

Después de cambiar el workspace, verifica:

```bash
# Debe mostrar: C:\Users\exclu\erp-taller-saas
pwd

# Debe mostrar: main
git branch --show-current

# Debe mostrar el último commit de main
git log --oneline -1
```

## 📝 REGLA PARA CURSOR AI

**SIEMPRE verificar el directorio de trabajo antes de hacer cambios:**

```bash
pwd  # Debe ser: C:/Users/exclu/erp-taller-saas
git branch --show-current  # Debe ser: main
```

**NUNCA trabajar en worktrees a menos que se especifique explícitamente.**
